⚡ Fast fix pattern for C video players

Most “opens then closes” issues are fixed by ensuring:

while(running){
    handle_events();
    render_frame();
}

If that loop is missing or exits on first frame → instant close.

🧪 Add debugging guard (VERY useful)

Wrap main like:

#include <stdio.h>
#include <stdlib.h>

int main(int argc,char*argv[]){
    printf("[JNET] Video player starting...\n");

    if(argc<2){

    }

    printf("[DEBUG] Opening file: %s\n",argv[1]);

    // init checks

        printf("[ERROR] Video init failed\n");

    }

    printf("[DEBUG] Entering main loop...\n");

        update_video();
        render();
    }

    printf("[DEBUG] Exit clean\n");
    return 0;

🐍 2. Python Qt crash (your real issue)

This is the actual failure:

AttributeError: 'MainWindow' object has no attribute '_video_widget'
💥 Root cause

You are doing:

self._player.setVideoOutput(self._video_widget)

BUT _video_widget was never created before that line.

🔧 Fix pattern (correct Qt order)

You MUST ensure this order:

self._setup_ui()
self._setup_video_widget()
self._setup_qtmedia()
✅ Minimal correct fix

Inside __init__:

self._video_widget=QVideoWidget(self)
self._setup_qtmedia()

OR better:

def _setup_ui(self):
    self._video_widget=QVideoWidget(self)
